2. Arquitectura del Sistema (Backend en Render)
Servidor Backend: El motor de procesamiento de lenguaje natural está alojado en Render.

Framework: FastAPI versión 0.109.0 para la creación de la API REST.

Entorno de Ejecución: Python 3.12.7 gestionado mediante runtime.txt.

Endpoints Principales: * GET /: Health check que verifica el estado de la API y la carga de la base de conocimiento local.

POST /chat: Procesa las consultas de los usuarios utilizando la lógica RAG.

3. Implementación de Inteligencia Artificial (RAG)
Orquestador: LangChain versión 0.1.0 para la gestión de cadenas de recuperación y memoria.

Modelo de Lenguaje: GPT-3.5-Turbo de OpenAI con una temperatura de 0.7 para equilibrar la precisión y la fluidez.

Base de Datos Vectorial: ChromaDB (v0.4.22) para el almacenamiento persistente de embeddings.

Procesamiento de Documentos:

Carga: Utiliza TextLoader para leer el archivo Conocimiento.txt.

Fragmentación (Chunking): El texto se divide en fragmentos de 500 caracteres con un solapamiento (overlap) de 50 caracteres para no perder el contexto semántico entre bloques.

Recuperación: Se utiliza una cadena RetrievalQA que extrae los 3 fragmentos más relevantes para generar cada respuesta.
